# Shift Game Jam 项目设计方案与后续功能补全

> 更新时间：2026-04-18  
> 目标：总结当前已落地架构、明确开发约束，并给出下一阶段功能补全路线。

---

## 1. 项目目标与玩法定位

- 题材：2D 平台跳跃解谜（Game Jam 主题：**Shift**）。
- 核心循环：玩家在“看似同一张地图”的多个关卡中反复通关。
- 关卡差异：每关 `Shift` 键能力不同（冲刺/巨化/重力反转/风力/换位等）。
- 关卡组织：使用 `BaseLevel` 作为母场景，子关卡继承并微调节点。

---

## 2. 当前架构总览（已实现）

## 2.1 全局事件中心（AutoLoad）

文件：`scripts/autoload/SignalManager.gd`

- `signal level_completed(next_level_id: int)`
- `signal player_died()`
- `signal shift_ability_used(ability_name: String)`

职责：
- 统一跨场景广播关卡完成、玩家死亡、能力使用事件。
- 降低 Player / Level / Hazard 等模块间硬引用。

## 2.2 玩家模块

核心文件：`scripts/player/Player.gd`

已实现要点：
- 基础横版移动、跳跃、重力、朝向翻转、落地反馈。
- 动画与音效状态驱动（待机/走/跑/起跳/下落/落地/死亡）。
- 死亡流程：`die()` -> 播放死亡表现 -> 最终发 `SignalManager.player_died`。
- 能力入口与解耦：通过 `ShiftHandler`/能力接口进行切换，不在 Player 内写死具体能力逻辑。
- 跳跃手感增强：已加入 **Coyote Time**。
  - `@export var coyote_time: float = 0.15`
  - `_coyote_timer` 落地重置、离地递减
  - 起跳后强制 `_coyote_timer = 0.0` 防二段跳

## 2.3 Shift 能力系统

目录：`scripts/abilities/`

- 基类：`ShiftAbility.gd`
- 具体能力：
  - `Ability_Sprint.gd`
  - `Ability_Giant.gd`
  - `Ability_GravityFlip.gd`
  - `Ability_Wind.gd`
  - `Ability_Swap.gd`

设计原则：
- 能力逻辑与 Player 运动逻辑解耦。
- Player 暴露受控接口（如倍率、重力状态、风力等），能力只调用接口不直接改底层流程。

## 2.4 关卡控制与继承

核心文件：`scripts/level/LevelController.gd`

已实现要点：
- `@export var starting_ability: Resource`：在关卡 Inspector 指定开局能力。
- 玩家生成与定位（`PlayerSpawn`）。
- 目标点 `Goal` 触发 `level_completed`。
- 监听 `player_died` 后延时重开当前场景。

关卡组织：
- 母场景：`scenes/levels/BaseLevel.tscn`
- 子关卡示例：`scenes/levels/Level_05_Swap.tscn`

## 2.5 危险物（尖刺）模块

文件：
- 预制体：`scenes/hazards/Spike.tscn`
- 逻辑：`scripts/hazards/Spike.gd`

已实现要点：
- 尖刺使用独立 `Area2D` 预制体，便于在 TileMap 中批量摆放。
- 信号连接全部代码化：`_ready()` 内自动连接 `body_entered`。
- 基于分组验证身份：仅当 `body.is_in_group("player")` 时触发死亡。
- 保护机制：启动宽限帧 + `overlaps_body` 二次确认，降低场景重载瞬间误触发概率。

## 3. 当前开发约束（必须遵守）

1. **Godot 4.3+ 规范优先**
- 地图层使用 `TileMapLayer`（不回退旧 `TileMap` 流程）。

2. **模块解耦**
- Player 不承担具体关卡能力逻辑。
- Hazard 不依赖节点名硬编码，统一基于 Group 和事件。

3. **关卡继承方案固定**
- 共同结构在 `BaseLevel`。
- 具体关卡只做最小覆盖（能力、布局、机关开关）。

4. **信号优先而非硬耦合调用链**
- 统一通过 `SignalManager` 做全局事件流转。

5. **编辑器手工连线风险控制**
- 关键触发（如 Spike）优先代码连线，避免场景复制后丢线。

---

## 4. 现阶段已知风险与注意事项

1. **碰撞层/掩码配置风险**
- `Player`、`Spike`、`Terrain` 的 layer/mask 必须一致约定，否则会出现“看见碰到但不触发”或“未接触误触发”。

2. **关卡继承覆盖风险**
- 子场景覆盖父节点属性后，可能导致 group/monitoring/collision 参数与母场景不一致。

3. **重开时序风险**
- 死亡动画、信号广播、场景重载存在时序关系，需避免重复重入和抖动重载。
