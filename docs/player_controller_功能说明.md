# player_controller.gd 功能说明

`player_controller.gd` 原本承担的是玩家角色的基础控制与表现层职责，后续这些机能已经合并进 `scripts/player/Player.gd`。

## 原脚本负责的内容

### 1. 基础移动

- 读取左右输入，控制玩家水平移动。
- 在没有输入时，对 `velocity.x` 做缓停处理。
- 在地面按下跳跃键时立即起跳。
- 玩家离地后持续施加重力。

### 2. 朝向处理

- 根据左右移动方向切换朝向。
- 同步翻转 `AnimatedSprite2D` 与备用 `Sprite2D`。

### 3. 动画状态切换

- 待机时播放 `default`。
- 行走时播放 `walk`。
- 高速移动时播放 `run`。
- 上升阶段播放 `jump_up`。
- 下落阶段播放 `jump_down`。
- 落地缓冲时播放 `jump_end`。
- 死亡时播放 `die`。

### 4. 落地与跳跃衔接

- 记录上一帧是否在地面，用于识别“刚落地”的时机。
- 落地时触发落地音效。
- 如果角色资源中存在 `jump_end` 动画，则先播放落地收尾动画，再回到待机。

### 5. 音效驱动

- 行走时播放循环步行音效。
- 冲刺时播放循环奔跑音效。
- 跳跃时播放跳跃音效。
- 落地时播放落地音效。
- 在动画状态切换时，适时停止循环类移动音效。

### 6. 死亡表现

- 进入死亡状态后停止水平移动。
- 清空速度并停止移动相关音效。
- 播放死亡动画。

## 与现有 Player.gd 的对应关系

现在这些功能已经迁移到 `scripts/player/Player.gd`，并在此基础上做了扩展：

- 保留了移动、跳跃、重力、朝向、动画和音效逻辑。
- 合并了原 `Player.gd` 中的死亡判定与能力接口。
- 死亡流程升级为：先停止移动并播放 `die` 动画，动画结束后再发出 `player_died` 信号。
- Shift 机能从主脚本中拆分到了 `scripts/player/player_shift_handler.gd`，方便独立切换模式和后续扩展。

## 为什么删除该文件

删除 `player_controller.gd` 的原因是避免以下问题：

- 玩家逻辑分散在两个脚本中，维护成本高。
- 场景引用和 uid 容易出现双份来源，导致后续整理困难。
- Shift、动画、死亡流程耦合在旧控制器与新玩家脚本之间，不利于继续扩展。

现在工程里玩家唯一主入口应为：

- `scripts/player/Player.gd`
- `scenes/entities/Player.tscn`
